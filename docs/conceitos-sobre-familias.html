Bem, as famílias por sua vez são divididas em três espécies:

  • System - Famílias de sistema. Exemplos: Wall, Floor, Ceiling.

  • Loadable - Famílias carregáveis. Exemplos: Door, Window, Furniture.

  • Model-in-place - As famílias "modeladas no local" são espécies de famílias modeladas diretamente no projeto e que serão usadas apenas naquele projeto.

Element Type

A classe ElementType, representa os tipos de famílias de sistema no Revit.

Diz na documentação " Base class for all types of elements in Autodesk Revit " (Classe base para "todos" os tipos de elementos no Autodesk Revit). Porém, como dissemos anteriormente, os Element Types são apenas os tipos de famílias de sistema.

Family Symbol

A classe FamilySymbol representa os tipos de famílias carregáveis e também os tipos de famílias Model-in-place no Revit.

Veja que na sessão Remarks da documentação diz em outras palavras o que falamos anteriormente em Conceitos Básicos da API do Revit. Sendo que precisamos atentar que o Remarks da imagem acima, refere-se a Famílias Carregáveis.

Family Instance

A classe FamilyInstance representa as instâncias de famílias carregáveis e também as instâncias de famílias Model-in-place no Revit.

Model-in-place

Perceba que:

A API do Revit não tem classes específicas para os tipos e instâncias Model-in-place.

Isso significa que as instâncias Model-in-place são tratadas da mesma forma que as instâncias de famílias carregáveis. De modo que não há um meio direto para identificar se uma instância é Model-in-place ou não.

Existem várias maneiras de contornar isso, porém as mais confiáveis são:

  1. Para comparar se uma instância que deveria ser de sistema é Model-in-place ou não, use o método Python isinstance() para comparar se a instância é da classe de sistema que você espera.

All Elements of Category

Este é um Node que retorna todas as instâncias que estão na categoria selecionada.

Isso significa que ele retorna todas as instâncias de famílias de sistema, carregáveis e Model-in-place, sem fazer qualquer distinção.

Veremos que usando a API do Revit existem meios diferentes para coletar instâncias de famílias de sistema e carregáveis.
